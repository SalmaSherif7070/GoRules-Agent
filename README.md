# Rule-to-GoRules Compiler Agent (MCP + Gemini)

Converts business rules + table schemas into executable GoRules decision tables
using Gemini as the compiler engine, exposed as a REST API and MCP server.
Generated rule files can be viewed and edited in the GoRules UI.

```
Rules CSV ──► Schema Inference ──► Gemini Compiler ──► GoRules JSON ──► Validation Result
                                         ▲                    │
                                     MCP Tools          GoRules UI (localhost:3000)
```

## Setup

```bash
# 1. Create virtual environment
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure environment
cp .env.example .env
# Edit .env — set GEMINI_API_KEY
```

### .env reference

| Variable             | Default                  | Description                     |
|----------------------|--------------------------|---------------------------------|
| `GEMINI_API_KEY`     | *(required)*             | Google AI Studio key            |
| `GEMINI_MODEL`       | `gemini-2.5-flash`       | Gemini model name               |
| `GEMINI_MAX_TOKENS`  | `4096`                   | Max output tokens               |
| `GEMINI_TEMPERATURE` | `0.1`                    | Low = deterministic output      |
| `MCP_SERVER_HOST`    | `0.0.0.0`                | Server bind host                |
| `MCP_SERVER_PORT`    | `8000`                   | Server port                     |
| `TABLES_DIR`         | `data/tables`            | Path to table CSV directory     |
| `RULES_FILE`         | `data/rules/rules.csv`   | Path to rules CSV               |
| `LOG_LEVEL`          | `INFO`                   | Logging verbosity               |

## Running

### 1. Start the Python server

```bash
python main.py
# REST API at http://localhost:8000
# Swagger docs at http://localhost:8000/docs
```

### 2. Generate GoRules JSON files from rules.csv

Call this once (or whenever rules change):

```bash
curl -X POST http://localhost:8000/api/generate-gorules
```

Or use Swagger: `http://localhost:8000/docs` → `/api/generate-gorules` → Try it out → Execute.

This reads all 20 rules from `data/rules/rules.csv`, calls Gemini once per rule,
and writes the output to `GoRules_rules/rule_{id}/Script_1.json`.

### 3. Fix edge format (one-time, required for GoRules UI)

The GoRules UI requires `"type": "edge"` on every edge object.
Run this once after generating:

```bash
python fix_edges.py
```

## Viewing Rules in the GoRules UI

### Step 1 — Start the GoRules editor via Docker

```bash
docker run -d -p 3000:3000 \
  -v "${PWD}/GoRules_rules:/app/data/gorules_rules" \
  --name gorules-ui \
  gorules/editor
```

### Step 2 — Copy rule files into the container

```bash
# macOS / Linux
docker cp GoRules_rules/. gorules-ui:/app/data/

# Windows PowerShell
docker cp GoRules_rules/. gorules-ui:/app/data/
```

### Step 3 — Open the UI

```
http://localhost:3000
```

Your rule files appear in the file tree on the left. Click any rule to open the
decision table editor. Use the Simulate panel to test with custom input values.

### Docker management

```bash
docker stop gorules-ui      # Stop
docker start gorules-ui     # Restart
docker logs gorules-ui      # Debug
docker rm gorules-ui        # Remove (need to re-run docker run next time)
```

## REST API Endpoints

### Validation

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/validate` | Validate INSERT or UPDATE against all rules |
| `GET`  | `/api/health` | Check server status + loaded counts |

### Tables & Rules

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET`  | `/api/tables` | List all loaded table names |
| `GET`  | `/api/tables/{name}` | Preview schema + sample rows |
| `GET`  | `/api/rules` | List all 20 business rules |

### GoRules Generation

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/generate-gorules` | Generate JSON files for all rules |

### GoRules UI Integration

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET`  | `/api/decisions` | List all generated decision files |
| `GET`  | `/api/decisions/{key}` | Load one decision JSON |
| `PUT`  | `/api/decisions/{key}` | Save editor changes back to disk |
| `POST` | `/api/decisions/{key}/simulate` | Run Zen engine with test input |

## MCP Tools (Claude Desktop)

Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "gorules-compiler": {
      "command": "python",
      "args": ["/absolute/path/to/gorules-mcp/main.py", "--stdio"],
      "env": {
        "GEMINI_API_KEY": "your_key_here"
      }
    }
  }
}
```

Available tools:

| Tool | Description |
|------|-------------|
| `validate_operation_tool` | Compile rules + validate INSERT or UPDATE |
| `list_tables_tool` | List all loaded table names |
| `get_table_tool` | Preview schema + sample rows |
| `upload_table_tool` | Upload a new table CSV (base64) |
| `list_rules_tool` | List all business rules |
| `upload_rules_tool` | Upload a new rules CSV (base64) |

## Example: Validate a Salary Increase

**Request:**

```json
POST /api/validate
{
  "operation": "UPDATE",
  "target_table": "employees",
  "target_row": {
    "employee_id": 4,
    "salary": 8500,
    "previous_salary": 7000,
    "performance_rating": 3
  },
  "previous_row": {
    "employee_id": 4,
    "salary": 7000,
    "previous_salary": 6500,
    "performance_rating": 3
  }
}
```

**Response:**

```json
{
  "operation_valid": false,
  "violated_rules": [
    {
      "rule_id": "1",
      "reason": "Salary increase (21.4%) exceeds maximum 20% — max allowed: 8400"
    }
  ],
  "changed_fields": ["salary"],
  "rules_evaluated": 20
}
```

## Adding a New Domain

1. Drop your CSVs into `data/tables/`
2. Replace `data/rules/rules.csv` with your rules
3. Call `POST /api/generate-gorules` — Gemini compiles them automatically
4. Run `python fix_edges.py`
5. Copy into Docker and refresh the UI

No code changes needed.
